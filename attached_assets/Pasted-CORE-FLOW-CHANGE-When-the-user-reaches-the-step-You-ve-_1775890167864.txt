SIZE-BASED SYSTEM (IMPORTANT UPDATE)
CORE IDEA:

The bounding box is used to represent a real-world measurement system for the T-shirt.

REAL WORLD MAPPING:

I already have a real T-shirt sizing system, for example:

👉 Small T-shirt = 50cm × 70cm (real physical size)

SYSTEM BEHAVIOR:

When the user selects a size (Small / Medium / Large):

The bounding box must represent that real-world size in DATA ONLY
IMPORTANT:

❌ Do NOT visually resize the box
❌ Do NOT move the box
❌ Do NOT apply any scaling in UI

WHAT ACTUALLY HAPPENS:

Example:

If user selects:

Size: Small

Then internally:

Bounding box = 50 × 70 (real-world cm equivalent)

If user selects Large:

Bounding box values change accordingly (real-world size)

BUT:

👉 The box stays visually identical on screen
👉 Only numeric values change behind the scenes

PURPOSE OF THIS SYSTEM:

This bounding box is used so that:

👉 When I place an image inside the box
👉 I can calculate its real print size relative to the actual T-shirt

So I can know:

How big the design is on a real Small / Medium / Large T-shirt
Exact physical scaling for printing
